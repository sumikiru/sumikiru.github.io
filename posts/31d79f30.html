其他插件配置

Leetcode 插件

名字为”Leetcode with labuladong”，个人认为比插件 Leetcode Editor 好用一些（最主要是设置文件名称的问题，Leetcode Editor 总是保存不了自定义配置）。
安装以后侧边栏能看到，按照说明走不多说，登录的时候需要用到 cookie，这里跟着官方文档走即可：登录Leetcode账号。

插件设置

之后进入设置->工具->Leetcode with labuladong，进行以下修改：

  • Website: leetcode.cn（美国站就选另一个）
  • CodeType: C++
  • FilePath：看你希望放在哪里，但至少保证这个文件夹下存在 CMakeList.txt，能够运行 cpp 文件。例如 FilePath 为 E:\Study\Algorithm，则力扣文件会放在 E:\Study\Algorithm\leetcode 路径下
  • HtmlContent: 推荐勾选
  • labuladongCookie：同样填登录时用到的 cookie
  • Code FileName: 建议改为 $!{question.frontendQuestionId}-${question. titleSlug}，这样能保证不含中文名，运行或调试不会报错，例如 1-two-sum.cpp

Single File Execution

插件全名为”C/C++ Single File Execution”，用户单个 cpp 文件的运行，可以很好解决前面所说的问题。
鼠标右键可以看到选项 add executable for single c/cpp file，点击后重新加载 CMake 项目（如未自动变化请手动右键文件根目录），它会自动在 CMakeLists.txt 末尾加上 addexecutable() 。

杂项

一些其他推荐使用的插件：

  • CodeGlance Pro
  • Key Promoter X
  • Nested Brackets Colorer
  • One Dark Theme
  • Atom Material Icons
  • Competitive Programming Helper（仅限 vscode）

vscode 配置 C++，需要以下插件：

  • C/C++
  • clangd
  • Clang-Format
  • Code Runner
  • Code Debugger
  • CodeLLDB
